# 서울 라스트 (MySeoul) — 게임잼 프로토타입

서울에 홀로 남은 생존자가 **한정된 4×4 가방에 모양 있는 아이템을 어떻게 패킹하느냐**로
7일간의 생존과 엔딩이 갈리는 모바일 분기 게임 (60초류).
선택지를 고르는 게 아니라 **가방 배치가 곧 선택**이다.

## 핵심 루프
기본 소지품 3개로 시작 → 서울 지도에서 지역 클릭 → 이벤트(가방 아이템 유무로 판정)
→ 결과(아이템 획득/체력 피해) → 가방 정리(패킹 퍼즐) → 7일 반복 → 엔딩 분기

- **가방**: 4×4 격자, 아이템마다 모양(도끼=ㄱ자, 우산=1×3 등). 아이템은 이벤트로 획득되어 가방이 점점 참 → 패킹 압박 증가.
- **판정**: 아이템 유무(presence).
- **지도**: 서울 25개 구 중 7개(종로·마포·영등포·용산·강남·송파·노원)만 활성.
- **엔딩**: 생존 / 사망 / 무전기 보유 시 구조(히든).

## 실행
- Unity **6000.3.10f1**
- 씬: `Assets/Scenes/SampleScene.unity` → Play
- 씬의 `Game` 오브젝트에 붙은 `GameManager`가 UI를 런타임 생성한다.

## 구조 (`Assets/Scripts/`)
| 파일 | 역할 |
|------|------|
| `GameData.cs` | 아이템/이벤트/가방 모델 (순수 C#) |
| `SeoulMap.cs` | 서울 25개 구 좌표 데이터 |
| `UIFactory.cs` | 코드 UI 생성 헬퍼 (한글: 맑은 고딕 동적 로드) |
| `DraggableItem.cs` | 드래그·격자 스냅·충돌 판정 |
| `GameManager.cs` | 상태머신 + 런타임 UI 빌드 |

설계 문서: `docs/specs/2026-06-25-seoul-apocalypse-prototype-design.md`

## 알려진 한계 (프로토타입)
- 아이템 회전 미구현 (모양 고정)
- 지도는 임시 도형판 (실제 지도 이미지 교체 예정)
- 아트는 단색 블록 + 텍스트로 대체
