# 서울 라스트 (가제) — 게임잼 프로토타입 설계

작성일: 2026-06-25
엔진: Unity 6000.3.10f1 / 프로젝트 `D:/Unity/MySeoul`
목적: 게임잼 전 **하루 만에** 만드는 플레이 가능한 프로토타입. 팀에게 핵심 "느낌"을 전달하는 것이 유일한 목표.

## 한 줄 컨셉
서울에 홀로 남은 생존자가, **한정된 4×4 가방에 무엇을 어떻게 욱여넣느냐**로 7일간의 생존과 엔딩이 갈리는 모바일 분기 게임.

## 핵심 기둥 (셀링포인트)
1. **가방 패킹이 곧 선택** — 선택지 클릭이 아니라, 모양 있는 아이템(도끼=ㄱ자, 우산=1×3, 라이터=1×1)을 4×4 격자에 배치한다.
2. **아이템은 이벤트로 획득됨** → 가방이 점점 찬다 → **패킹 압박이 날마다 심해진다** (긴장 곡선).
3. **판정은 유무(presence)** — "도끼 있음 → 통과 / 없음 → 피해". 단순·명확.

## 게임 루프
```
기본 소지품 3개로 시작 (가방에 배치)
  → 서울 지도에서 활성 지역 클릭 (7개 중, 미방문 지역 택1)
  → 이벤트 발생 → 가방 아이템 유무 + 체력으로 판정
  → 결과 (아이템 획득 / 체력 피해 텍스트)
  → 가방 정리 타임 (배치 퍼즐: 새 아이템 배치/기존 폐기)
  → 다음 지역 선택 …  (7일 반복)
  → 누적 상태(생존/체력/핵심템)에 따라 엔딩 분기
```

## 무대 / 데이터
- **지도**: 서울 25구 중 7구만 활성, 나머지 비활성(프로토에선 7개 버튼으로 표현 가능).
  활성 7구: 종로 · 마포 · 영등포 · 용산 · 강남 · 송파 · 노원. 하루 하나씩 방문, 방문하면 잠김.
- **스탯**: **체력(HP)** 1종만. 시작 100, 이벤트 실패 시 감소, 0이면 사망 엔딩.
- **가방**: 4×4 격자(16칸). 아이템은 셀 모양(shape)을 가짐. 모양 충돌 시 배치 불가.
- **이벤트**: 지역당 1개(총 7개). 각 이벤트는 "필요 아이템 id 유무"로 성공/실패 2갈래.
- **엔딩**: 3종 — ① 7일 생존 탈출(생존) ② 체력 0 사망 ③ 특정 핵심템 보유 시 히든(예: 무전기 → 구조 요청 성공).

## 아키텍처 (에이전트 구현 친화적)
> 핵심 결정: **UI를 씬에서 손으로 배선하지 않고 코드에서 런타임 생성**한다.
> 단일 `GameBootstrap` MonoBehaviour가 Canvas/패널/그리드/버튼을 코드로 만든다.
> → MCP 기반 작업에서 수십 번의 수동 참조 연결을 피하고, 재현·리뷰가 쉬움.

씬 구성: `SampleScene`에 GameObject 하나(`Game`) + `GameBootstrap` 컴포넌트 + EventSystem(코드 생성). 카메라는 기본 유지.

스크립트(`Assets/Scripts/`):
- **데이터/모델 (순수 C#, MonoBehaviour 아님)**
  - `ItemShape` — 셀 오프셋 목록(예: 도끼 = (0,0),(0,1),(1,1)). 회전 지원(90° 단위).
  - `ItemDef` — id, 표시명, 모양, 색(아이콘 대용).
  - `ItemDatabase` — 하드코딩 아이템 풀.
  - `BagModel` — 4×4 점유 배열. `CanPlace/Place/Remove/Contains(id)`.
  - `EventDef` / `EventDatabase` — 지역별 이벤트(필요 itemId, 성공/실패 텍스트, 보상 itemId, 체력 변화).
  - `GameState` — day, hp, 방문지역, 가방 참조.
- **흐름**
  - `GameManager` — 상태 머신: Pack → Map → Event(결과) → Organize → (반복) → Ending.
- **뷰 (런타임 UI 생성 + 입력)**
  - `UIBuilder` — 공통 UI 팩토리(패널/버튼/텍스트/이미지 생성 헬퍼).
  - `BagView` — 4×4 격자 렌더 + 아이템 오브젝트 생성.
  - `DraggableItem` — IBeginDrag/IDrag/IEndDrag, 격자 스냅, 회전 버튼.
  - `MapView` — 7지역 버튼, 방문 시 비활성.
  - `EventView` — 이벤트 텍스트/결과 표시.
  - `HudView` — Day/HP 표시.
  - `EndingView` — 엔딩 텍스트 + 재시작.

## 하루 프로토타입 MVP 우선순위
1. **가방 4×4 + 모양 패킹 드래그** ← 정체성. 시간 몰빵.
2. 지도 7지역 클릭 → 이벤트 텍스트.
3. 유무 판정 → 결과 → 가방에 보상 아이템 추가.
4. 7일 카운터 + 체력 + 엔딩 3종.

> 시간 부족 시 지도는 버튼 7개로 충분. 가방 패킹만큼은 절대 빼지 않는다(게임 정체성).

## 비목표 (YAGNI)
- 조합/시너지 판정(추후 확장), 다중 스탯, 실제 서울 지도 SVG, 사운드/연출, 세이브, 인벤토리 무게.
- 아트는 단색 사각형 + 텍스트로 대체. 느낌 전달이 목적.
