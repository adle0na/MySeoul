# 인게임 화면 재설계 — 사이드스크롤 Explore 연출

작성일: 2026-06-27
프로젝트: `D:/Unity/MySeoul`
성격: GameFlow(대화그래프·스탯·인벤·스테이지/루프)는 데이터/상태 엔진으로 유지하고, **인게임 표현 레이어를 사이드스크롤 연출로 교체.** 진행 규칙은 Day 카운터/장소 표기만 추가.

## 화면 구성 (위→아래)
1. **상단바**: `DAY n · 현재 장소` + 설정 기어(자리만). Day = (탐색 5 + 휴식) 1바퀴마다 +1. 장소 = 현재 스테이지 지역.
2. **좌측 상태 레일(버티컬)**: 배고픔/갈증/아픔/정신병 4개. 평소 숨김, **30%↑ 주의(노랑)·60%↑ 위험(빨강·경고)**. 내부 스탯 인덱스(허기/수분/건강/기운=배고픔/갈증/아픔/정신병)에 매핑. 아이콘: `Assets/UI/icon/sangtea-isan-*`.
3. **중앙 씬**: 배경(`Assets/art/Backgrond/Bg_ex*`) 인피니티 스크롤(RawImage uvRect) + Spine 걷기(`Character_Yeger_walk`, SkeletonGraphic) 캐릭터가 좌측에서 걷기. 우측에서 아이템이 다가와 만나면 걷기 정지 → 이벤트.
4. **이벤트 카드(하단)**: 만남 시 제목+상황+선택지(3030frame). "가방에 넣자" 선택 시 아이템이 하단 가방 버튼으로 날아가는 트윈.
5. **하단 가방 토글 버튼**(open/close): 누르면 가방 그리드 오버레이(bg_inven). 사용 버튼은 기존 단일 사용 버튼.
6. **결과**: 선택 결과(= 다음 대화 노드의 텍스트/상태변화) 표시 → 확인 시 걷기 재개 → 다음 탐색.

## 한 탐색 사이클
```
걷기(배경 스크롤) → 아이템 등장·접근 → 만남(정지) → 이벤트 카드(대화 description+choices)
  → 선택 → (가방에 넣자=아이템 가방으로 / NewState 상태변화) → 결과 → 확인 → 다음 탐색
5탐색 → 휴식 → 지도 → 다음 Day
```
기존 대화 그래프의 description/choices/NextEventId/지급(DialogItemId)/NewState를 그대로 흘려보냄.

## 컴포넌트
- `ExploreView`: 상단바/상태레일/씬/이벤트카드/하단바 빌드·갱신 (GameFlow 내 BuildUI 확장)
- `ScrollBg`: 인피니티 배경(uvRect.x), 걷기 중에만 스크롤
- `WalkChar`: Spine SkeletonGraphic 걷기/정지
- `ItemApproach`: 아이템 우→좌 접근, 가방으로 이동 트윈
- `StatusRail`: 4상태 경고 표시(30/60)
- 스프라이트/배경/Spine 참조는 FlowCanvasBuilder(에디트 모드)가 `Assets/UI`·`Assets/art`에서 이름으로 할당(직렬화).

## 진행 규칙 변경
- `Day`: 스테이지(=탐색5+휴식) 완료마다 +1. 상단 표기.
- `현재 장소`: stageRegion 표기(온보딩은 "체육창고" 등 기본값).

## 상태 4종
- 표시명 배고픔/갈증/아픔/정신병, 임계치 **주의 30 / 위험 60** (기존 40/70 → 30/60).

## 이번 범위 밖
- 설정 메뉴 실제 동작, 사운드, 상태이상 "N일 지속" 타이머, 정교한 컷 연출.

## 성공 기준
걷기 → 아이템 만남 → 선택 → 결과 → (가방 이동) → 다음 탐색 → 휴식까지 한 바퀴가 연출로 끊김 없이 도는 것.
